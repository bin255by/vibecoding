⚙️ 技术实现细节（含代码）
🔍 1. 意图解析（非阻塞扫描）
watch(() => isAiTyping.value, (newVal) => {
if (!newVal) parseIntent(aiContent.value);
});

function parseIntent(text) {
requestIdleCallback(() => {
const keywords = ['小轮','巴士','铜锣湾'];
activeTools.value = keywords.filter(kw => text.includes(kw));
}, { timeout: 1000 });
}

使用 requestIdleCallback 空闲扫描，保证60fps，降级用 setTimeout。

目标：在AI回复完成后，扫描文本中的关键词（如“天星小轮”“铜锣湾”），决定是否显示快捷图标。 技术细节： 监听AI状态：使用watch监听一个布尔值isAiTyping，当变为false时触发扫描。 扫描函数：用正则或includes检测关键词，将匹配到的工具存入响应式数组activeTools。 性能优化：使用requestIdleCallback或setTimeout延迟执行，避免阻塞UI渲染。如果支持Web Worker，可以将扫描逻辑放到Worker中，但简单场景下requestIdleCallback足够。

💾 5. 行程看板 + Pinia持久化
// store/travel.js
export const useTravelStore = defineStore('travel', {
state: () => ({ savedItineraries: [] }),
actions: {
saveItinerary(it) {
this.savedItineraries.push({ id: Date.now(), ...it });
localStorage.setItem('saved', JSON.stringify(this.savedItineraries));
},
loadFromStorage() {
const saved = localStorage.getItem('saved');
if (saved) this.savedItineraries = JSON.parse(saved);
}
}
});
// 应用启动时调用 loadFromStorage

目标：将AI规划的行程转换为可保存的卡片，并存入“我的”页面。 技术细节： 数据结构设计：行程步骤数组，每个步骤包含地点、交通方式、时间等。 解析AI回复：如果后端返回结构化数据最好；否则前端需用正则提取。简单场景可假设后端已提供JSON。 动态组件渲染看板：在AI回复下方插入一个自定义组件，接收行程数据

❓ 预想面试问答
🔹 技术类
  • Q: requestIdleCallback 是什么？ A: 浏览器空闲时执行任务，不影响动画/交互，降级用 setTimeout。
  • Q: 组件复用如何保证自动搜索？ A: 同时监听 onMounted 和 route.query 的 watch。
  • Q: Pinia 和 Vuex 区别？ A: Pinia 无 mutations，TS 支持好，多 store 友好。
  • Q: 数据持久化怎么做？ A: localStorage + Pinia 初始化读取，或用插件 persistedstate。
  • Q: 地点匹配不到怎么办？ A: 相似度阈值控制，低于阈值只展示通用搜索按钮。
  • Q: query 和 params 区别？ A: query 显示在 URL（可分享），params 不显示（刷新丢失）。
  • Q: 滚动流畅性？ A: scrollIntoView({ behavior: 'smooth' }) + 先展开卡片。
