🧠 技术实现分步解析（Vue 功底展示）

如果面试官问：“你打算怎么具体实现这些功能？” 请按以下步骤展示：

1. 意图解析：非阻塞式文本扫描

做法 在对话组件中监听 AI 状态。当 isStreaming 变为 false 时，使用正则匹配内容中的关键词。

技术点 使用 Web Worker 或 requestIdleCallback 进行扫描，确保解析过程不占主线程，保证聊天界面 60 帧不卡顿。

// 在 Vue 组件中监听 AI 状态 watch(() => isAiTyping.value, (newVal) => { if (!newVal) { // AI 说完了 parseIntent(aiContent.value); } }); function parseIntent(text) { // 定义关键词映射 const map = { '小轮': 'ferry', '巴士': 'bus', '铜锣湾': 'metro' }; // 识别文本中包含的关键词并存入响应式数组 activeTools.value = Object.keys(map).filter(key => text.includes(key)); }

2. 动态组件：Icon 栏的“插队”逻辑

做法 定义一个响应式数组 activeActions。

技术点 利用 Vue 的 v-for 渲染。扫描到关键词后，将对应的 Config（含 Icon、Label、Path）推入数组，实现 Icon 栏的动态更新。

<!-- 在快捷按钮栏组件中 --> <div class="action-bar"> <div v-for="tool in activeTools" :key="tool" @click="handleJump(tool)"> <img :src="getIcon(tool)" /> <span>{{ tool }}详情</span> </div> </div>

3. 跨页面参数传递：路由与全局状态

做法 巴士搜索：调用 router.push({ path: '/bus-search', query: { keyword: '中环' } })。港铁详情：通过 Pinia 存入 targetStationId: 8。

技术点 利用 Vue Router 的导航守卫（beforeRouteEnter）或 onMounted 钩子读取数据。

// 第一步：AI 界面点击跳转 function handleJump(tool) { router.push({ path: '/bus-search', query: { keyword: '1路巴士', autoSearch: true } }); } // 第二步：巴士搜索页接收数据 (BusSearch.vue) onMounted(() => { const { keyword, autoSearch } = route.query; if (keyword && autoSearch) { this.searchVal = keyword; // 自动填表 this.fetchData(); // 自动触发搜索方法 } });

4. 自动触发交互：实现“跳过去就看到结果”

做法 在巴士搜索页：挂载后检测到 query.keyword，立即触发一次 fetchSearchList()。在港铁列表页：检测到 stationId，使用 el.scrollIntoView() 定位，并改变该项的 isExpanded 状态，触发蓝底展开效果。

💡 面试加分：终极 QA 准备

  • 问：为什么选择 Pinia 而不是 Vuex？
    答：Pinia 对 TypeScript 支持更友好，API 更简洁（没有 mutations），且完美契合 Vue 3 的 Composition API 风格，性能更轻量。
  • 问：如果 AI 提到的地点在数据库搜不到怎么办？
    答：我会设计“模糊匹配权重”。匹配度低则显示通用搜索按钮，不进行错误填充。
  • 问：Vue 的性能怎么保证？
    答：使用 markRaw 优化静态配置，避免 Vue 对无需变动的数据进行深层响应式追踪；同时利用路由懒加载减少首屏 JS 体积。

🗣️ 第三部分：面试官可能会问你的问题（提前准备）

问：如果 AI 提到的地点，我们数据库里匹配不到怎么办？

答：设置一个“相似度权重”。如果匹配度低于 80%，就只展示一个通用的搜索按钮，不自动填充，避免误导用户。我们可以设置一个匹配权重，如果匹配度不高，就只展示通用的搜索入口，不进行自动填充，保证准确性。

问：频繁跳转会不会让用户觉得迷失方向？

答：我会使用 Vue 的 <Transition> 动画或者 iOS 原生的交互动效，并提供清晰的“返回 AI 助手”按钮，让用户能随时回到对话上下文。我们可以加上平滑的转场动画，并在目标页面顶部保留明显的“返回 AI 助手”按钮，保持上下文的连贯。

问：为什么用 Vue 做这个有优势？

答：因为 Vue 的响应式系统和组件化设计非常适合处理这种“数据驱动 UI”的场景。特别是 Teleport 或动态组件，可以非常轻量地实现这种交互联动。

问：Vue 的性能怎么保证？

答：我会使用 markRaw 优化那些不需要响应式的静态配置数据，并利用路由懒加载减少首屏压力。

🔍 面试官可能会追问的 Pinia 细节

问：Pinia 怎么实现持久化（刷新页面数据不丢）？

答：可以使用插件 pinia-plugin-persistedstate，或者在 Action 里手动把数据存入 localStorage，在 state 初始化时再读取出来。

问：为什么不用 storeToRefs？

答：如果直接解构 Store（如 const { id } = travelStore），数据会失去响应式。如果需要解构，必须使用 storeToRefs(travelStore)，但在你的场景中，直接通过 travelStore.id 访问是最稳妥且简单的。

问：Pinia 和 Vuex 的主要区别是什么？

答：Pinia 舍弃了复杂的 Mutations，同步和异步统一放在 Actions 里处理，代码更简洁；同时它对 TypeScript 的类型推断是原生的，非常友好。
